## 2. 현재 버그: WP 접근 시 거리 증가 문제

### 2.1 증상

- WP02를 향해 **직선으로 걸어가는데** HUD 표시 거리가 `15m → 18m → 22m`로 **증가**
- 방향은 올바르게 가리키나, 걸을수록 목표에서 멀어지는 것으로 계산됨
- **앵커 1개만 인식**된 상태에서 발생 (0개 또는 2개+ 에서는 미발생)

### 2.2 재현 조건

1. 이전 세션에서 `AutoCalibrateFromAnchors()`가 heading 180°를 계산하여 PlayerPrefs 저장
2. 앱 재시작 → 앵커 1개만 인식 (Auto calibrate 실패, 2개 미만)
3. `LoadRoute()`가 `PlayerPrefs.GetFloat("ARNav_HeadingOffset")` → **180°** 로드
4. `ComputeMapCalibration()`이 heading=180° 적용 → `cos(-π) = -1` → **Z축 반전**
5. 참가자가 +Z 방향으로 걸으면 → 변환 후 -Z → WP에서 멀어짐

### 2.3 관련 코드

#### 2.3.1 `LoadRoute()` — heading 우선순위 결정

```csharp
public void LoadRoute(string routeId)
{
    // Route B 고정 — 단일 경로 사용
    activeRoute = routeB;
    CurrentWaypointIndex = 0;

    var cam = XRCameraHelper.GetCamera();
    if (cam != null)
        playerTransform = cam.transform;
    else
        Debug.LogWarning("[WaypointManager] LoadRoute: 카메라를 찾을 수 없어 playerTransform 미설정");

    // SpatialAnchorManager에서 앵커 Transform 바인딩
    BindAnchorTransforms(routeId);

    // SLAM→도면 좌표 변환 초기 계산 (heading 보정 전)
    ComputeMapCalibration();

    // ★ Heading 보정: auto → stored → fallback 우선순위
    bool autoCalibrated = AutoCalibrateFromAnchors();
    if (autoCalibrated)
    {
        headingSource = "auto";
        ComputeMapCalibration();  // heading 변경 → 맵 보정 재계산
    }
    else if (PlayerPrefs.HasKey("ARNav_HeadingOffset"))
    {
        // ★★★ 버그 원인: 이전 세션의 180°가 여기서 로드됨
        headingCalibrationOffset = PlayerPrefs.GetFloat("ARNav_HeadingOffset");
        headingSource = "stored";
        ComputeMapCalibration();
    }
    else
    {
        headingCalibrationOffset = fallbackHeadingOffset;  // 현재 0°
        headingSource = "fallback";
        ComputeMapCalibration();
    }

    Debug.Log($"[WaypointManager] Heading offset applied: " +
        $"{headingCalibrationOffset:F1}° (source={headingSource})");
    DomainEventBus.Instance?.Publish(
        new HeadingCalibrationApplied(headingSource, headingCalibrationOffset));

    PublishRouteBindingSummary(routeId);
    lastSnapshotTime = Time.time;
    cachedArrowRenderer = FindObjectOfType<ARArrowRenderer>();

    int nearestIdx = FindNearestWaypointIndex();
    if (nearestIdx > 0)
    {
        CurrentWaypointIndex = nearestIdx;
        Debug.Log($"[WaypointManager] Start index adjusted to {nearestIdx} " +
            $"({activeRoute.waypoints[nearestIdx].waypointId})");
    }
}
```

> **핵심**: `auto` (앵커 2+) → `stored` (PlayerPrefs) → `fallback` (0°) 순서. 앵커 1개일 때 auto 실패 → stored가 우선 → **이전 세션의 잘못된 값이 적용**.

#### 2.3.2 `AutoCalibrateFromAnchors()` — 2개 미만 시 실패

```csharp
private bool AutoCalibrateFromAnchors()
{
    if (activeRoute == null) return false;

    var anchored = new List<(Vector3 fallback, Vector3 slam)>();
    foreach (var wp in activeRoute.waypoints)
    {
        if (wp.anchorTransform != null)
            anchored.Add((wp.fallbackPosition, wp.anchorTransform.position));
    }
    if (anchored.Count < 2) return false;  // ★ 1개면 여기서 return false

    // 가장 먼 쌍 선택 (정밀도 향상)
    float maxDist = 0f;
    int bestI = 0, bestJ = 1;
    for (int i = 0; i < anchored.Count; i++)
        for (int j = i + 1; j < anchored.Count; j++)
        {
            float d = Vector3.Distance(anchored[i].slam, anchored[j].slam);
            if (d > maxDist) { maxDist = d; bestI = i; bestJ = j; }
        }

    Vector3 slamDir = anchored[bestJ].slam - anchored[bestI].slam;
    Vector3 mapDir = anchored[bestJ].fallback - anchored[bestI].fallback;
    slamDir.y = 0; mapDir.y = 0;
    if (slamDir.sqrMagnitude < 0.01f || mapDir.sqrMagnitude < 0.01f) return false;

    float slamYaw = Mathf.Atan2(slamDir.x, slamDir.z) * Mathf.Rad2Deg;
    float mapYaw = Mathf.Atan2(mapDir.x, mapDir.z) * Mathf.Rad2Deg;
    headingCalibrationOffset = slamYaw - mapYaw;

    // ★ 자동 보정 성공 시 PlayerPrefs에 저장 → 다음 세션에서 stored로 로드
    PlayerPrefs.SetFloat("ARNav_HeadingOffset", headingCalibrationOffset);
    PlayerPrefs.Save();
    return true;
}
```

#### 2.3.3 `ComputeMapCalibration()` — heading 180° 시 Z축 반전

```csharp
private void ComputeMapCalibration()
{
    var pairs = GetAllCalibratedAnchorPairs();

    if (pairs.Count == 0)
    {
        // 앵커 0개 → 카메라 시작 위치를 WP01 fallback에 매핑
        if (playerTransform == null || activeRoute == null ||
            activeRoute.waypoints.Count == 0)
        {
            hasMapCalibration = false;
            calibrationSource = "none";
            return;
        }

        int startIdx = activeRoute.waypoints.Count > 1 ? 1 : 0;
        var startWP = activeRoute.waypoints[startIdx];
        var cameraXZ = new Vector2(playerTransform.position.x,
                                    playerTransform.position.z);
        var fallbackXZ = new Vector2(startWP.fallbackPosition.x,
                                      startWP.fallbackPosition.z);

        // ★★★ heading=180° → theta=-π → cos=-1, sin≈0
        float theta = -headingCalibrationOffset * Mathf.Deg2Rad;
        mapCalibCos = Mathf.Cos(theta);  // cos(-π) = -1
        mapCalibSin = Mathf.Sin(theta);  // sin(-π) ≈ 0

        float rotX = cameraXZ.x * mapCalibCos - cameraXZ.y * mapCalibSin;
        float rotY = cameraXZ.x * mapCalibSin + cameraXZ.y * mapCalibCos;
        mapCalibOffset = fallbackXZ - new Vector2(rotX, rotY);

        hasMapCalibration = true;
        calibrationSource = "auto_zero_anchor";
        return;
    }

    if (pairs.Count == 1)
    {
        // ★ 앵커 1개 + stored heading → 동일한 cos/sin 반전 문제
        float theta = -headingCalibrationOffset * Mathf.Deg2Rad;
        mapCalibCos = Mathf.Cos(theta);
        mapCalibSin = Mathf.Sin(theta);

        var (slamXZ, fallbackXZ) = pairs[0];
        float rotX = slamXZ.x * mapCalibCos - slamXZ.y * mapCalibSin;
        float rotY = slamXZ.x * mapCalibSin + slamXZ.y * mapCalibCos;
        mapCalibOffset = fallbackXZ - new Vector2(rotX, rotY);
        calibrationSource = "anchor_1";
    }
    else
    {
        // 앵커 2+개: 최소자승법 (heading과 무관하게 자체 계산)
        int n = pairs.Count;
        Vector2 centroidS = Vector2.zero, centroidF = Vector2.zero;
        for (int i = 0; i < n; i++)
        {
            centroidS += pairs[i].slamXZ;
            centroidF += pairs[i].fallbackXZ;
        }
        centroidS /= n; centroidF /= n;

        float sumCross = 0f, sumDot = 0f;
        for (int i = 0; i < n; i++)
        {
            var ds = pairs[i].slamXZ - centroidS;
            var df = pairs[i].fallbackXZ - centroidF;
            sumCross += ds.x * df.y - ds.y * df.x;
            sumDot += ds.x * df.x + ds.y * df.y;
        }

        float theta = Mathf.Atan2(sumCross, sumDot);
        mapCalibCos = Mathf.Cos(theta);
        mapCalibSin = Mathf.Sin(theta);

        var rotatedCS = new Vector2(
            centroidS.x * mapCalibCos - centroidS.y * mapCalibSin,
            centroidS.x * mapCalibSin + centroidS.y * mapCalibCos);
        mapCalibOffset = centroidF - rotatedCS;
        calibrationSource = $"anchor_{n}";
    }
    hasMapCalibration = true;
}
```

#### 2.3.4 `SlamToFloorPlan()` — 좌표 변환

```csharp
private Vector2 SlamToFloorPlan(Vector2 slamXZ)
{
    float rx = slamXZ.x * mapCalibCos - slamXZ.y * mapCalibSin;
    float ry = slamXZ.x * mapCalibSin + slamXZ.y * mapCalibCos;
    return new Vector2(rx, ry) + mapCalibOffset;
}
```

> heading=0° 일 때 `cos=1, sin=0` → `SlamToFloorPlan(p) = p + offset` (정상 평행이동)
> heading=180° 일 때 `cos=-1, sin≈0` → `SlamToFloorPlan(p) = (-p.x, -p.y) + offset` (**부호 반전**)

#### 2.3.5 `ComputeDistanceToWaypoint()` — fallback 거리 계산

```csharp
private float ComputeDistanceToWaypoint(Waypoint wp, out float arrivalRadius)
{
    if (wp.anchorTransform != null)
    {
        // 앵커 존재: SLAM 직접 거리
        var wpPos = wp.anchorTransform.position;
        arrivalRadius = wp.radius;
        return Vector3.Distance(
            new Vector3(playerTransform.position.x, 0, playerTransform.position.z),
            new Vector3(wpPos.x, 0, wpPos.z));
    }
    else if (hasMapCalibration)
    {
        // ★ Fallback: SlamToFloorPlan() 사용 → heading 180°면 거리 반전
        var playerXZ = new Vector2(playerTransform.position.x,
                                    playerTransform.position.z);
        var playerFloorPlan = SlamToFloorPlan(playerXZ);
        var wpXZ = new Vector2(wp.fallbackPosition.x, wp.fallbackPosition.z);
        arrivalRadius = wp.radius * fallbackRadiusMultiplier;  // 2.5배
        return Vector2.Distance(playerFloorPlan, wpXZ);
    }
    else
    {
        arrivalRadius = wp.radius;
        return float.MaxValue;
    }
}
```

#### 2.3.6 `GetDistanceToNext()` — HUD 표시값

```csharp
public float GetDistanceToNext()
{
    if (playerTransform == null || activeRoute == null ||
        CurrentWaypointIndex >= activeRoute.waypoints.Count)
        return 0f;

    var wp = activeRoute.waypoints[CurrentWaypointIndex];
    if (wp.anchorTransform != null)
    {
        return Vector3.Distance(playerTransform.position,
                                wp.anchorTransform.position);
    }
    else if (hasMapCalibration)
    {
        // ★ 동일한 SlamToFloorPlan() 경로 → HUD에 반전된 거리 표시
        var playerXZ = new Vector2(playerTransform.position.x,
                                    playerTransform.position.z);
        var playerFloorPlan = SlamToFloorPlan(playerXZ);
        var wpXZ = new Vector2(wp.fallbackPosition.x, wp.fallbackPosition.z);
        return Vector2.Distance(playerFloorPlan, wpXZ);
    }
    else
    {
        return -1f;  // 보정 불가
    }
}
```

## 3. 근본 원인 분석

### 3.1 PlayerPrefs 잔존값 시나리오

```
[세션 1] 앵커 5개 인식
  └→ AutoCalibrateFromAnchors() = true
  └→ heading = 180° (SLAM/도면 정반대 방향)
  └→ PlayerPrefs.SetFloat("ARNav_HeadingOffset", 180) ← 저장

[세션 2] 앵커 1개만 인식 (재인식 불안정)
  └→ AutoCalibrateFromAnchors() = false (앵커 < 2)
  └→ PlayerPrefs.HasKey("ARNav_HeadingOffset") = true
  └→ headingCalibrationOffset = 180°  ← ★ 이전 세션 값 로드
  └→ headingSource = "stored"

[세션 2에서 걸을 때]
  플레이어가 +Z로 1m 이동
  └→ SlamToFloorPlan: cos(-180°)=-1 → Z 좌표 반전
  └→ 도면 좌표계에서 -Z로 1m 이동한 것으로 계산
  └→ WP02(Z=33)에서 2m 멀어짐 (기대: 2m 가까워짐)
```

### 3.2 수학적 검증

heading=180° 일 때의 `SlamToFloorPlan()` 동작:

```
theta = -180° × π/180 = -π
cos(theta) = cos(-π) = -1
sin(theta) = sin(-π) ≈ 0

SlamToFloorPlan(x, z):
  rx = x × (-1) - z × 0 = -x
  ry = x × 0 + z × (-1) = -z
  result = (-x, -z) + offset
```

즉, SLAM 좌표의 **X와 Z가 모두 부호 반전**된다. 플레이어가 WP 방향(+Z)으로 걸으면, 변환 후에는 -Z로 이동한 것으로 계산되어 **거리가 증가**.

### 3.3 앵커 개수별 보정 동작

| 앵커 수 | heading 소스 | ComputeMapCalibration 분기 | 결과 |
|--------|-------------|--------------------------|------|
| 0개 | stored (180°) | `pairs.Count == 0` → zero-anchor | **Z축 반전 (버그)** |
| 1개 | stored (180°) | `pairs.Count == 1` → 단일 앵커 | **Z축 반전 (버그)** |
| 2+개 | auto (재계산) | `pairs.Count >= 2` → 최소자승법 | 정상 (heading 무관) |

> **핵심**: 2개 이상일 때는 `AutoCalibrateFromAnchors()`가 새 heading을 계산하고, `ComputeMapCalibration()`의 `pairs >= 2` 분기가 자체적으로 θ를 계산하므로 stored heading의 영향을 받지 않음. **0~1개일 때만** stored heading이 그대로 적용되어 문제 발생.

### 3.4 추가 위험 영역

PlayerPrefs `"ARNav_HeadingOffset"`를 저장하는 위치가 **5곳** 존재:

| 위치 | 메서드 | 트리거 |
|------|--------|--------|
| 1 | `AutoCalibrateFromAnchors()` | 앵커 2+ 인식 시 자동 |
| 2 | `AdjustHeadingOffset(float delta)` | ExperimenterHUD ±5° 버튼 |
| 3 | `ManualCalibrateAtCurrentWaypoint()` | ExperimenterHUD 수동 보정 |
| 4 | `SetHeadingCalibrationOffset(float)` | ImageTrackingAligner 마커 인식 |
| 5 | `LoadRoute()` | 저장 안 함 (읽기만) |

모든 저장 위치가 동일 키를 사용하므로, **마지막으로 저장된 값이 다음 세션에서 무조건 로드**됨. 특히 실험 중 `AdjustHeadingOffset()`으로 미세 조정한 값이 다음 참가자에게 적용되는 위험도 있음.

```
수정 후보:
A) PlayerPrefs 삭제: 세션 시작 시 "ARNav_HeadingOffset" 키 삭제
   → 항상 fallback(0°)부터 시작, auto가 성공하면 덮어씀
   → 장점: 가장 안전, 이전 세션 오염 완전 차단

B) 참가자별 키: "ARNav_HeadingOffset_{pid}" 사용
   → 장점: 참가자 간 오염 방지
   → 단점: 같은 참가자의 이전 세션 오염은 여전히 가능

C) stored 값 범위 검증: |stored| > 90° 면 무시
   → 장점: 극단적 반전만 차단
   → 단점: 정당한 180° 회전 환경에서 동작 불가

D) 앵커 개수 조건 추가: pairs < 2이면 stored 무시, fallback 사용
   → 장점: 1개 앵커 시나리오에서 안전
   → 단점: stored가 정확했던 경우 정밀도 손실
```

**권장 방안**: **A + 세션 초기 PlayerPrefs 클리어**. 이유:

1. 현재 설계에서 stored heading이 유효한 시나리오가 거의 없음 (같은 세션 내에서는 auto가 이미 작동)
2. 참가자 간 오염 방지가 실험 신뢰도에 가장 중요
3. 구현 1줄 (`PlayerPrefs.DeleteKey("ARNav_HeadingOffset")` in `LoadRoute()` 시작부)
