## 1. 프로젝트 개요

### 1.1 연구 제목

**불확실한 실내 길찾기에서 스마트글래스-스마트폰 하이브리드 사용이 기기 전환 행동과 신뢰 보정에 미치는 영향**

### 1.2 핵심 개념

스마트 글래스(XReal Air2 Ultra)와 스마트폰(XReal Beam Pro)을 조합한 **하이브리드 실내 내비게이션** 시스템에서, 두 기기의 상보적 역할 분리를 통해 사용자의 교차검증 행동과 시스템 신뢰 보정을 연구한다.

```
글래스 (WHERE to go)          Beam Pro (WHAT to know)
· AR 방향 화살표               · 인터랙티브 맵 + POI 마커
· 실시간 경로 안내              · POI 상세 정보 카드
· Head-up, Hands-free          · 상황 정보 카드 / 비교 카드
                               · 미션 참조 패널
         ↕ 상황에 따라 전환 ↕
```

### 1.3 이론적 프레임워크

#### 분산 인지 (Distributed Cognition)

Hutchins(1995)의 분산 인지 이론에 따르면, 인지 과정은 개인 내부에만 국한되지 않고 환경의 도구와 표상에 걸쳐 분산된다.

- **Glass Only 조건**: 인지 자원이 하나의 기기-사용자 인터페이스에 집중 (hands-free + head-up, 제한된 정보 표현)
- **Hybrid 조건**: 인지 과정이 두 기기에 걸쳐 분산 → 사용자가 상황에 따라 정보 소스를 전환하는 **적응적 인지 분산**

#### 신뢰 보정 (Trust Calibration)

Lee & See(2004)의 신뢰 프레임워크에 따르면, 자동화 시스템에 대한 신뢰는 시스템의 수행과 사용자의 기대 간 일치도에 의해 보정된다.

- **신뢰 위반**: 불확실성 트리거(모호한 교차로, 화살표 소실 등)가 시스템 신뢰를 저하시키는 이벤트
- **보정 기제**: Hybrid 조건에서 대안 기기를 통한 교차 검증이 신뢰 보정을 촉진
- **과보정 vs 적정 보정**: Glass Only에서는 대안 없이 신뢰가 과도하게 저하될 수 있음

#### 두 프레임워크의 통합

```
┌──────────────────────────────────────────────────────────────┐
│              분산 인지 (Distributed Cognition)                │
│                                                              │
│  글래스 (Ambient — WHERE)  ←──전환──→  Beam Pro (Focal — WHAT)│
│   · 방향 화살표                    · 인터랙티브 맵 + POI 마커 │
│   · 실시간 안내                    · POI 상세 카드            │
│                                    · 상황 정보 카드           │
│                                    · 비교 카드 (미션 C)       │
│                                                              │
│          ↓ 불확실성 트리거 발생 시 ↓                          │
│                                                              │
│              신뢰 보정 (Trust Calibration)                    │
│                                                              │
│  단일 기기: 보정 정보 부재 → 신뢰 급락 또는 과신              │
│  하이브리드: 다양한 콘텐츠로 교차 검증 → 적응적 신뢰 보정     │
└──────────────────────────────────────────────────────────────┘
```

## 2. 연구 질문 & 가설

### 2.1 연구 질문 (RQ)

> **RQ**: 불확실한 실내 길찾기에서 스마트글래스-Beam Pro 하이브리드 사용이 글래스 단독 사용 대비 기기 전환 행동과 신뢰 보정에 미치는 영향은 무엇인가?

| 하위 질문 | 내용 |
|-----------|------|
| **RQ-a** | Hybrid 조건에서 참가자가 Beam Pro 정보 허브를 참조하는 패턴(빈도, 시점, 지속시간, 콘텐츠 유형별 접근 분포)은 어떠한가? |
| **RQ-a'** | 어떤 유형의 Beam Pro 콘텐츠(POI 상세, 상황 카드, 비교 카드, 맵)가 미션 수행 정확도와 가장 강한 연관을 보이는가? |
| **RQ-b** | 기기 조건에 따라 불확실성 지점에서의 신뢰 보정(trust calibration)은 어떻게 달라지는가? |
| **RQ-c** | 기기 조건에 따라 길찾기 수행 효율(시간, 정지)은 어떻게 달라지는가? |

### 2.2 가설

#### H1: 신뢰 보정 (Trust Calibration)

> Hybrid 조건에서 확신도-정확도 상관(calibration)이 Glass Only 조건보다 높다. 즉, Hybrid 사용자는 자신의 확신도가 실제 미션 수행 결과와 더 잘 일치한다.

| 측정 | 지표 |
|------|------|
| 1차 | 확신도-미션 정확도 상관계수 (calibration index) — 참가자별 피어슨 r |
| 2차 | 트리거 전후 확신도 변화량 |
| 3차 | 사후 시스템 신뢰 척도 (Jian et al. 7문항) |

통계: Paired t-test (또는 Wilcoxon signed-rank)

#### H2: 수행 효율 (Task Performance)

> Hybrid 조건은 Glass Only 대비 미션 정확도가 높다. 단, 과제 완료 시간은 Beam Pro 참조로 인해 동등하거나 약간 길 수 있다.

| 측정 | 지표 |
|------|------|
| 1차 | 미션 정확도 — 전체 및 미션 타입별 (A/B/C) 정답률 |
| 2차 | 총 과제 완료 시간 (초) |
| 3차 | 정지 횟수 및 총 정지 시간 |
| 4차 | 미션별 소요시간 |

#### H3: 교차검증 행동 (Cross-Verification)

> Hybrid 조건에서 트리거 발생 시 Beam Pro 참조 확률이 비트리거 구간 대비 유의하게 높으며, Beam Pro 참조는 미션 정확도 향상과 정적 상관을 보인다.

| 측정 | 지표 |
|------|------|
| 1차 | 교차검증 지수(CVI) — 트리거 구간 참조율 ÷ 전체 참조율 |
| 2차 | Beam Pro 참조 후 정확도 vs 미참조 시 정확도 |
| 3차 | 콘텐츠별 CVI (POI/카드/비교/맵 각각) |
| 4차 | 정보 활용도 — 콘텐츠 유형별 열람 후 미션 정확도 상관 |

## 3. 실험 설계

### 3.1 설계 요약

| 항목 | 내용 |
|------|------|
| **설계** | 2 (기기 조건) × Within-subjects |
| **기기 조건** | Glass Only (G) / Hybrid (H) |
| **참가자 수** | 24명 (2순서 × 12명) |
| **균형화** | 단순 역균형 (G→H, H→G) |
| **경로 수** | 2개 (조건당 1개), 단일 층 |
| **소요 시간** | 참가자당 약 50분 |

### 3.2 조건별 기기 구성

| | Glass Only | Hybrid |
|---|-----------|--------|
| **글래스 출력** | AR 화살표 + BeamPro WorldSpace | AR 화살표 |
| **Beam Pro** | 화면 잠금 (접근 불가) | 정보 허브 활성 (lift-to-wake) |
| **핸드트래킹** | 활성 (Hand Ray) | 비활성 (터치 조작) |
| **정보 접근** | 화살표 + 환경 관찰만 | 화살표 + 환경 + 정보 허브 |

**핵심 비교 논리**: 두 조건의 유일한 차이는 Beam Pro 정보 허브 접근 가능 여부

```
Glass Only:   글래스(WHERE: 방향 화살표) ─────────────→ 미션 수행
Hybrid:       글래스(WHERE: 방향 화살표)
              + Beam Pro(WHAT: 정보 허브) ────────────→ 미션 수행
                         ↑
                  이것의 효과를 측정
```

### 3.3 카운터밸런싱

**조건 순서** (`CounterbalanceConfig.asset`):

| 순서 그룹 | 1차 조건 | 2차 조건 |
|-----------|---------|---------|
| **S1** | Glass Only | Hybrid |
| **S2** | Hybrid | Glass Only |

**경로 순서** (참가자 번호로 결정):

| 참가자 번호 | 1차 경로 | 2차 경로 |
|------------|---------|---------|
| **홀수** (P01, P03, ...) | Route A | Route B |
| **짝수** (P02, P04, ...) | Route B | Route A |

균형화 검증: 조건×순서 12:12, 경로×순서 12:12, 경로-조건 4조합×각 6회 ✓

## 4. 미션 & 트리거 설계

### 4.1 미션 타입 (3종)

| 타입 | 명칭 | 수/경로 | 요구 인지 | Beam Pro 추가 가치 |
|------|------|---------|----------|-------------------|
| **A** | 방향 안내 + 검증 | 2 | 내비게이션 + 환경 관찰 | POI 상세로 빠르고 정확한 검증 |
| **B** | 모호한 의사결정 | 2 | 불확실성 하의 의사결정 | 교차로 주변 POI + 상황 카드로 정보 기반 판단 |
| **C** | 정보 통합 | 1 | 기기 정보 + 환경 정보 교차 참조 | 비교 카드로 효율적 정보 통합 |

### 4.2 미션 배치 (경로당 5개: A1→B1→A2→B2→C1)

```
경로 시작점
    │
    ├── WP01: 미션 A1 브리핑 → 이동 → WP02: 미션 A1 검증
    │
    ├── WP03: [트리거] + 미션 B1 (모호한 갈림길)
    │
    ├── WP04: 미션 A2 브리핑 → 이동 → WP05: 미션 A2 검증
    │
    ├── WP06: [트리거] + 미션 B2 (불확실한 구간)
    │
    ├── WP07: 미션 C1 (정보 통합) → WP08: 미션 C1 검증
    │
경로 종점
```

### 4.3 불확실성 트리거 (4종)

| 트리거 | 명칭 | 메커니즘 | 의도 |
|--------|------|----------|------|
| **T1** | 안내 열화 | AR 화살표 6초 떨림 → 3초 소실 → 자동 복구 | 시스템 신뢰 위반, 대안 기기 참조 동기 |
| **T2** | 정보 불일치 | 화살표 방향 ≠ 환경 표지판 방향 (화살표가 정답) | 다중 정보원 갈등, 교차검증 행동 유발 |
| **T3** | 해상도 부족 | 화살표가 ±30° 부채꼴로 퍼짐 (미션 종료까지) | 글래스 정보 한계 체감, Beam Pro 지도 필요성 |
| **T4** | 안내 부재 | "목적지 근처입니다"만 표시, 화살표 제거 (미션 종료까지) | 자력 탐색, Beam Pro 지도 가치 극대화 |

**트리거 배치**:

| | 경로 A | 경로 B |
|---|--------|--------|
| **WP03 부근** | T1 (안내 열화) | T2 (정보 불일치) |
| **WP06 부근** | T4 (안내 부재) | T3 (해상도 부족) |

각 참가자는 2개 경로에서 트리거 4회 경험 (4종 각 1회)

## 8. 분석 계획

### 8.1 가설별 분석 방법

| 가설 | 주 분석 | 보조 분석 |
|------|--------|----------|
| **H1** (신뢰 보정) | 대응표본 t-test — 조건별 calibration index 비교 | 트리거 전후 확신도 변화량, 사후 신뢰 척도 비교 |
| **H2** (수행 효율) | 대응표본 t-test — 미션 정확도, 완료 시간 | 미션 타입별 정확도 비교 (반복측정 ANOVA: 조건 × 미션 타입) |
| **H3** (교차검증) | 기술통계 + 카이제곱 — CVI, 트리거/비트리거 참조율 비교 | 피어슨 상관 — Beam Pro 참조 ↔ 미션 정확도 |

비정규성 시 비모수 대안: Wilcoxon signed-rank test

### 8.2 탐색적 분석

- **미션 타입별 기기 전환 패턴**: 미션 A/B/C에서 Beam Pro 참조 빈도·시간 비교
- **트리거 유형별 반응 차이**: T1~T4 각각의 확신도 변화·반응시간
- **검증 행동 분류**: 사전적(proactive) vs 반응적(reactive) Beam Pro 참조
- **학습 효과**: 1차→2차 경로에서 기기 전환 전략 변화
- **콘텐츠 접근 패턴 분류**: 맵 중심 / 카드 중심 / 혼합형
- **정보 과부하 징후**: Beam Pro 접근량 ↔ NASA-TLX 상관, 역U자형 관계 여부

### 8.3 분석 스크립트 (4개)

```bash
python3 analysis/analyze_device_switching.py   # 전환 패턴, CVI
python3 analysis/analyze_trust_performance.py  # 신뢰, 확신도, calibration
```
