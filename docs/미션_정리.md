# 미션 정리

## 개요

경로당 5개 미션, 총 10개 미션. 실행 순서: **A1 → B1 → A2 → B2 → C1**

### 미션 타입 (3종)

| 타입 | 설명 | 미션 수 |
|------|------|---------|
| **A_DirectionVerify** | 목적지 방향 확인 후 호실 번호 검증 | Route당 2개 (A1, A2) |
| **B_AmbiguousDecision** | 불확실성 트리거 발동 구간에서 방향 판단 | Route당 2개 (B1, B2) |
| **C_InfoIntegration** | 두 공간을 비교하여 정보 통합 판단 | Route당 1개 (C1) |

### 불확실성 트리거 (4종)

| 트리거 | 타입 | 설명 | 사용 미션 |
|--------|------|------|-----------|
| **T1** | TrackingDegradation | AR 화살표 떨림(6초) + 일시 소실(3초) | Route A - B1 |
| **T2** | InformationConflict | 물리적 표지판과 AR 안내 간 정보 충돌 (시스템 개입 없음) | Route B - B1 |
| **T3** | LowResolution | AR 화살표가 넓은 부채꼴(30도)로 표시되어 방향 모호 | Route B - B2 |
| **T4** | GuidanceAbsence | AR 화살표 소실 + "목적지 근처" 텍스트만 표시 | Route A - B2 |

## 미션 FSM (MissionManager 내부 상태)

```
Idle → Briefing → Navigation → Arrival → Verification → ConfidenceRating → DifficultyRating → Scored
                                                              ↓
                                                     (정답 여부 기록)
```

각 미션마다 위 상태를 순환하며, 5개 미션 완료 후 `AllMissionsCompleted` 이벤트 발행.
