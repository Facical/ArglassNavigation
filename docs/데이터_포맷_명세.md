## 1. 이벤트 로그 CSV 스키마

### 1.1 컬럼 정의

| 컬럼명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| `timestamp` | ISO 8601 | 이벤트 발생 시각 (밀리초 포함) | `2026-03-15T14:32:01.234` |
| `participant_id` | string | 참가자 ID | `P01` |
| `condition` | enum | 기기 조건 | `GlassOnly` / `Hybrid` |
| `event_type` | enum | 이벤트 유형 (아래 정의 참조) | `WAYPOINT_REACHED` |
| `waypoint_id` | string | 관련 웨이포인트 ID (해당 시 / 없으면 빈값) | `WP03` |
| `head_rotation_x` | float | 머리 회전 X축 (pitch, 도) | `12.5` |
| `head_rotation_y` | float | 머리 회전 Y축 (yaw, 도) | `-45.2` |
| `head_rotation_z` | float | 머리 회전 Z축 (roll, 도) | `1.3` |
| `device_active` | enum | 현재 활성 기기 | `glass` / `beam_pro` / `both` / `none` |
| `confidence_rating` | int | 확신도 (1-7, 해당 시) | `5` |
| `mission_id` | string | 현재 미션 ID (해당 시) | `A1` |
| `difficulty_rating` | int | 미션 난이도 평정 (1-7, 해당 시) | `4` |
| `verification_correct` | bool | 검증 정답 여부 (해당 시) | `true` |
| `beam_content_type` | enum | Beam Pro 콘텐츠 유형 (해당 시) | `poi_detail` / `info_card` / `comparison` / `map` / `mission_ref` |
| `extra_data` | string (JSON) | 추가 데이터 (이벤트별 상이) | `{"offset_angle": 15}` |

## 2. 이벤트 타입 정의 (38종)

### 2.1 실험 제어 이벤트 (9종)

| 이벤트 타입 | 설명 | extra_data |
|------------|------|------------|
| `EXPERIMENT_START` | 실험 세션 시작 | `{"order_group": "S1"}` |
| `EXPERIMENT_END` | 실험 세션 종료 | `{"total_duration_s": 3200}` |
| `RELOCALIZATION_START` | 공간 앵커 재인식 시작 | — |
| `RELOCALIZATION_COMPLETE` | 재인식 완료 | `{"success_rate": 0.87, "success": 13, "timed_out": 2, "failed": 0, "total": 15}` |
| `RELOCALIZATION_PROCEED_PARTIAL` | 부분 재인식으로 진행 (실험자 결정) | `{"success_rate": 0.75, "fallback_waypoints": "WP03,WP06"}` |
| `RELOCALIZATION_RETRY` | 재인식 재시도 | — |
| `CONDITION_CHANGE` | 조건 전환 (GlassOnly ↔ Hybrid) | `{"condition": "GlassOnly"}` |
| `ROUTE_START` | 경로 수행 시작 | `{"mission_set": "Set1", "condition": "GlassOnly"}` |
| `SURVEY_START` | 설문 시작 | `{"survey": "post_condition_1"}` / `"post_condition_2"` / `"post_experiment"` |

> **미구현**: `ROUTE_END` — 코드에 해당 LogEvent 호출 없음. 대안: 마지막 `MISSION_COMPLETE` 시점 사용.

### 2.2 웨이포인트 이벤트 (3종)

| 이벤트 타입 | 설명 | extra_data |
|------------|------|------------|
| `WAYPOINT_REACHED` | 웨이포인트 도달 | waypointId 컬럼 사용 |
| `WAYPOINT_FALLBACK_USED` | 앵커 미추적, fallback 위치 사용 | `{"fallback_position": "(x,y,z)"}` |
| `WAYPOINT_LATE_ANCHOR_BOUND` | 백그라운드 복구로 앵커 지연 바인딩 | `{"drift_m": 1.2, "old_pos": "...", "new_pos": "..."}` |

> **미구현**: `WAYPOINT_SKIPPED` — 코드에 해당 LogEvent 호출 없음. 향후 구현 예정.

### 2.3 기기 전환 이벤트 (2종)

| 이벤트 타입 | 설명 | device_active / extra_data |
|------------|------|------------|
| `BEAM_SCREEN_ON` | Beam Pro 화면 켜짐 (하이브리드 조건 핵심) | device_active: `"beam_pro"` |
| `BEAM_SCREEN_OFF` | Beam Pro 화면 꺼짐 | device_active: `"glass"`, `{"duration_s": 4.2}` |

> **참고**: `BEAM_SCREEN_ON`~`BEAM_SCREEN_OFF` 쌍이 하이브리드 조건에서 기기 전환의 핵심 측정. `duration_s`는 ON부터 OFF까지의 시간.

### 2.4 Beam Pro 정보 허브 이벤트 (7종)

`BEAM_SCREEN_ON`~`BEAM_SCREEN_OFF` 사이에 발생하는 하위 이벤트. Beam Pro 정보 허브 내 상호작용을 상세 기록.

| 이벤트 타입 | 설명 | beam_content_type | extra_data |
|------------|------|-------------------|------------|
| `BEAM_TAB_SWITCH` | 정보 허브 탭 전환 | 전환 대상 탭 | `{"from_tab": "map", "to_tab": "info_card"}` |
| `BEAM_POI_VIEWED` | POI 상세 정보 열람 | `poi_detail` | `{"poi_id": "room_b125", "poi_type": "large_lecture"}` |
| `BEAM_INFO_CARD_OPENED` | 상황 정보 카드 열림 | `info_card` | `{"card_id": "a_card_01", "card_type": "sign_card", "auto_shown": true}` |
| `BEAM_INFO_CARD_CLOSED` | 상황 정보 카드 닫힘 | `info_card` | `{"card_id": "a_card_01", "view_duration_s": 5.2}` |
| `BEAM_MAP_ZOOMED` | 인터랙티브 맵 줌 조작 | `map` | `{"zoom_level": 2.5}` |
| `BEAM_COMPARISON_VIEWED` | 비교 카드 열람 | `comparison` | `{"comparison_id": "comp_a_c1"}` |
| `BEAM_MISSION_REF_VIEWED` | 미션 참조 패널 열람 | `mission_ref` | `{"mission_id": "A1", "ref_type": "briefing_review"}` |

> **참고**: `auto_shown` 필드는 카드가 시스템에 의해 자동 표시되었는지(true) 사용자가 직접 열었는지(false)를 구분. 자동 표시 카드의 열람율은 정보 허브 설계 유효성의 핵심 지표.

### 2.5 AR 표시 이벤트 (3종)

| 이벤트 타입 | 설명 | extra_data |
|------------|------|------------|
| `GLASS_ARROW_SHOWN` | 글래스에 방향 화살표 표시 | `{"angle": 45.2}` |
| `GLASS_ARROW_HIDDEN` | 글래스 화살표 숨김 | — |
| `GLASS_ARROW_OFFSET` | 불확실성 트리거: 화살표 오프셋 | T1: `{"offset_angle": 3.2, "trigger_id": "T1"}`, T3: `{"spread_angle": 30, "trigger_id": "T3"}` |

### 2.6 이동 이벤트

> **미구현**: `PAUSE_START` / `PAUSE_END` — 정지 감지 로직 미구현. 향후 2초 이상 정지 감지 기능 추가 시 사용.

### 2.7 미션 이벤트 (9종)

| 이벤트 타입 | 설명 | extra_data |
|------------|------|------------|
| `MISSION_START` | 미션 시작 (브리핑 표시) | `{"mission_id": "A1", "type": "A_DirectionVerify", "briefing": "..."}` |
| `MISSION_ARRIVAL` | 미션 목적지 도달 | `{"mission_id": "A1"}` |
| `VERIFICATION_ANSWERED` | 검증 질문 응답 | `{"mission_id": "A1", "answer": 2, "correct": true, "rt_s": 4.3}` |
| `CONFIDENCE_RATED` | 확신도 평정 (미션별) | `{"mission_id": "A1", "rating": 5}` |
| `MISSION_COMPLETE` | 미션 완료 (정답 판정 포함) | `{"mission_id": "A1", "correct": true, "duration_s": 95}` |
| `DIFFICULTY_RATED` | 미션 난이도 평정 | `{"mission_id": "A1", "rating": 4}` |
| `BRIEFING_FORCED` | 실험자가 브리핑 건너뛰기 (UI 상호작용 실패 시) | `{"mission_id": "A1", "forced": true}` |
| `ARRIVAL_FORCED` | 실험자가 도착 수동 선언 (fallback 모드 거리 체크 불가 시) | `{"mission_id": "A1", "forced": true}` |
| `MISSION_FORCE_SKIPPED` | 실험자가 미션 강제 건너뛰기 | `{"mission_id": "A1", "state": "Verification", "forced": true}` |

> `BRIEFING_FORCED`, `ARRIVAL_FORCED`, `MISSION_FORCE_SKIPPED`: 분석 시 `"forced":true` 플래그로 필터링 가능. 강제 이벤트가 포함된 미션은 유효성 검토 필요.
> `CONFIDENCE_RATED`: confidence_rating 컬럼에 1-7 값 기록. 미션 완료 직후 수집.
> `DIFFICULTY_RATED`: difficulty_rating 컬럼에 1-7 값 기록. 확신도 평정 후 수집.
> `VERIFICATION_ANSWERED`: verification_correct 컬럼에 정답 여부 기록.

### 2.8 트리거 이벤트 (2종)

| 이벤트 타입 | 설명 | extra_data |
|------------|------|------------|
| `TRIGGER_ACTIVATED` | 불확실성 트리거 활성화 | `{"trigger_type": "T1_TrackingDegradation", "trigger_id": "T1"}` |
| `TRIGGER_DEACTIVATED` | 트리거 비활성화 (복구) | `{"trigger_type": "T1_TrackingDegradation", "duration_s": 10.5}` |

> 트리거 유형 (TriggerType enum):
> - `T1_TrackingDegradation`: 안내 열화 (jitter 6초 + blackout 3초)
> - `T2_InformationConflict`: 정보 불일치 (물리 표지판, 화살표 변화 없음)
> - `T3_LowResolution`: 해상도 부족 (30° 부채꼴, 미션 종료까지)
> - `T4_GuidanceAbsence`: 안내 부재 (화살표 제거 + 텍스트, 미션 종료까지)

### 2.9 디버그 이벤트 (3종)

| 이벤트 타입 | 설명 | extra_data |
|------------|------|------------|
| `GLASS_CAPTURE_START` | 글래스 뷰 녹화 시작 | `{"file": "capture_20260302.mp4"}` |
| `GLASS_CAPTURE_STOP` | 글래스 뷰 녹화 종료 | `{"file": "capture_20260302.mp4"}` |
| `GLASS_CAPTURE_ERROR` | 글래스 녹화 오류 | `{"error": "create_failed"}` / `"videomode_failed"` / `"recording_start_failed"` / `"recording_stop_failed"` |

> 디버그 이벤트는 분석에서 제외. 녹화 기능은 실험자가 수동 시작/종료.

## 3. 파일 네이밍 규칙

### 3.1 이벤트 로그

```
{participantId}_{condition}_{missionSet}_{timestamp}.csv
```

| 요소 | 형식 | 예시 |
|------|------|------|
| `{participantId}` | P + 2자리 숫자 | `P01`, `P24` |
| `{condition}` | `GlassOnly` / `Hybrid` | `GlassOnly` |
| `{missionSet}` | `Set1` / `Set2` | `Set1` |
| `{timestamp}` | `yyyyMMdd_HHmmss` | `20260315_143201` |

**예시**: `P01_GlassOnly_Set1_20260315_143201.csv`

### 3.2 설문 데이터

```
surveys/
├── pre_survey.csv          # 사전 설문 (전 참가자 통합)
```

## 4. 설문 CSV 스키마

### 4.1 사전 설문 (pre_survey.csv)

```csv
participant_id,age,gender,nav_app_frequency,ar_experience,building_familiarity
P01,24,M,daily,none,1
```

| 컬럼 | 타입 | 값 범위 |
|------|------|---------|
| `participant_id` | string | P01-P24 |
| `age` | int | 정수 |
| `gender` | enum | M / F / Other |
| `nav_app_frequency` | enum | daily / weekly / monthly / rarely / never |
| `ar_experience` | enum | none / tried_once / occasional / regular |
| `building_familiarity` | int | 1-5 (1=처음, 5=매우 익숙) |

### 4.2 NASA-TLX (nasa_tlx.csv)

```csv
participant_id,condition,mental_demand,physical_demand,temporal_demand,performance,effort,frustration
P01,GlassOnly,12,3,8,15,10,6
P01,Hybrid,10,4,7,18,9,5
```

각 하위척도 값: 0-21 (NASA-TLX raw score)

### 4.3 시스템 신뢰 척도 (trust_scale.csv)

```csv
participant_id,condition,trust_q1,trust_q2,trust_q3,trust_q4,trust_q5,trust_q6,trust_q7,trust_mean
```

각 문항 값: 1-7 (7점 리커트)
