# 데이터 포맷 명세

## 1. 이벤트 로그 CSV 스키마

### 1.1 컬럼 정의 (24컬럼)

| 컬럼명 | 타입 | 설명 | 예시 |
|--------|------|------|------|
| `timestamp` | ISO 8601 | 이벤트 발생 시각 (밀리초 포함) | `2026-03-15T14:32:01.234` |
| `participant_id` | string | 참가자 ID | `P01` |
| `condition` | enum | 기기 조건 | `GlassOnly` / `Hybrid` |
| `event_type` | enum | 이벤트 유형 (아래 정의 참조) | `WAYPOINT_REACHED` |
| `waypoint_id` | string | 관련 웨이포인트 ID (해당 시 / 없으면 빈값) | `WP03` |
| `head_rotation_x` | float | 머리 회전 X축 (pitch, 도) | `12.5` |
| `head_rotation_y` | float | 머리 회전 Y축 (yaw, 도) | `-45.2` |
| `head_rotation_z` | float | 머리 회전 Z축 (roll, 도) | `1.3` |
| `device_active` | enum | 현재 활성 기기 | `glass` / `beam_pro` / `both` / `none` |
| `confidence_rating` | int | 확신도 (1-7, 해당 시) | `5` |
| `mission_id` | string | 현재 미션 ID (해당 시) | `A1` |
| `difficulty_rating` | int | 미션 난이도 평정 (1-7, 해당 시) | `4` |
| `verification_correct` | bool | 검증 정답 여부 (해당 시) | `true` |
| `beam_content_type` | enum | Beam Pro 콘텐츠 유형 (해당 시) | `poi_detail` / `info_card` / `comparison` / `map` / `mission_ref` |
| `session_id` | string | 세션 고유 ID | `P01_20260315_143201` |
| `mission_set` | enum | 미션 세트 | `Set1` / `Set2` |
| `trigger_id` | string | 트리거 ID (해당 시) | `T1` / `T2` / `T3` / `T4` |
| `trigger_type` | string | 트리거 유형 (해당 시) | `T1_TrackingDegradation` |
| `cause` | string | 이벤트 원인 | `radius` / `pass_through` / `beam_switch` |
| `duration_s` | float | 지속 시간 (초) | `10.5` |
| `distance_m` | float | 거리 (미터) | `3.45` |
| `anchor_bound` | bool | 앵커 바인딩 여부 | `true` / `false` |
| `arrow_visible` | bool | AR 화살표 표시 여부 | `true` / `false` |
| `extra_data` | string (JSON) | 추가 데이터 (이벤트별 상이, 하위 호환) | `{"offset_angle": 15}` |

### 1.2 CSV 헤더

```csv
timestamp,participant_id,condition,event_type,waypoint_id,head_rotation_x,head_rotation_y,head_rotation_z,device_active,confidence_rating,mission_id,difficulty_rating,verification_correct,beam_content_type,session_id,mission_set,trigger_id,trigger_type,cause,duration_s,distance_m,anchor_bound,arrow_visible,extra_data
```

### 1.3 전용 컬럼 매핑

이벤트별로 전용 컬럼에 값이 채워지는 규칙:

| 이벤트 | 전용 컬럼 | 값 |
|--------|----------|---|
| `DIFFICULTY_RATED` | `difficulty_rating` | 1-7 |
| `MISSION_COMPLETE` | `verification_correct`, `duration_s` | 정답 여부, 미션 소요 시간 |
| `WAYPOINT_REACHED` | `cause` | `radius` / `pass_through` / `look_ahead_skip` / `look_ahead_target` / `manual` |
| `TRIGGER_ACTIVATED` | `trigger_id`, `trigger_type` | T1~T4, TriggerType 문자열 |
| `TRIGGER_DEACTIVATED` | `trigger_id`, `trigger_type`, `duration_s` | 트리거 지속 시간 |
| `TRIGGER_INTERRUPTED` | `trigger_id`, `trigger_type`, `duration_s` | 트리거 중단 시점 지속 시간 |
| `TRIGGER_RESPONSE` | `trigger_id`, `trigger_type`, `duration_s`, `cause` | 반응 시간, 첫 번째 행동 |
| `BEAM_SCREEN_OFF` | `duration_s` | ON→OFF 지속 시간 |
| `NAV_STATE_SNAPSHOT` | `distance_m`, `anchor_bound`, `arrow_visible` | 타겟 거리, 앵커 상태, 화살표 상태 |
| `PAUSE_END` | `duration_s` | 정지 지속 시간 |
| `BEAM_POI_VIEWED` | `duration_s` | 열람 시간 |
| `BEAM_INFO_CARD_CLOSED` | `duration_s` | 카드 열람 시간 |

## 2. Sidecar 파일 스키마

이벤트 CSV와 동일한 세션 prefix를 공유하는 보조 파일들. 고빈도 연속 데이터와 세그먼트 데이터를 분리하여 이벤트 CSV의 간결성을 유지.

### 2.1 Head Pose Trace (`*_head_pose.csv`, 10Hz)

```csv
timestamp,session_id,participant_id,condition,mission_id,waypoint_id,yaw,pitch,roll,angular_velocity_yaw,beam_on,arrow_visible,trigger_id
```

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `yaw` | float | 머리 수평 회전 (도) |
| `pitch` | float | 머리 상하 회전 (도) |
| `roll` | float | 머리 좌우 기울기 (도) |
| `angular_velocity_yaw` | float | yaw 각속도 (°/s) |
| `beam_on` | bool | Beam Pro 활성 여부 |
| `arrow_visible` | bool | AR 화살표 표시 여부 |
| `trigger_id` | string | 현재 활성 트리거 (없으면 빈값) |

**분석 용도**: head scanning amplitude, search entropy, trigger 후 orientation recovery time

**예상 크기**: 30분 실험 기준 ~18,000행 (~2MB)

### 2.2 Navigation Trace (`*_nav_trace.csv`, 2Hz)

```csv
timestamp,session_id,mission_id,current_wp_index,current_wp_id,target_wp_id,player_x,player_y,player_z,target_x,target_y,target_z,distance_m,speed_ms,anchor_bound,is_fallback,has_map_calib,calib_source,heading_offset_deg,arrow_visible,beam_on,trigger_id
```

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `current_wp_index` | int | 현재 웨이포인트 인덱스 |
| `current_wp_id` | string | 현재 웨이포인트 ID |
| `target_wp_id` | string | 미션 타겟 웨이포인트 ID |
| `player_x/y/z` | float | SLAM 좌표계 플레이어 위치 |
| `target_x/y/z` | float | 현재 WP 타겟 위치 |
| `distance_m` | float | 타겟까지 거리 |
| `speed_ms` | float | 이전 샘플 대비 이동 속도 (m/s) |
| `anchor_bound` | bool | 현재 WP 앵커 바인딩 여부 |
| `is_fallback` | bool | fallback 좌표 사용 여부 |
| `has_map_calib` | bool | 맵 보정 활성화 여부 |
| `calib_source` | string | 보정 소스 |
| `heading_offset_deg` | float | heading 보정 오프셋 (도) |

**분석 용도**: path efficiency, hesitation zone dwell, decision-point stop time, waypoint overshoot, fallback-dependent drift

**예상 크기**: 30분 기준 ~3,600행 (~720KB)

### 2.3 Anchor Relocalization (`*_anchor_reloc.csv`)

```csv
timestamp,session_id,waypoint_id,anchor_guid,is_critical,state,elapsed_s,slam_reason,remap_attempt,file_exists,in_sdk
```

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `waypoint_id` | string | 앵커가 매핑된 웨이포인트 |
| `anchor_guid` | string | 앵커 GUID |
| `is_critical` | bool | 우선 로드 대상 여부 |
| `state` | enum | `Pending` / `InProgress` / `Tracking` / `TimedOut` / `LoadFailed` |
| `elapsed_s` | float | 경과 시간 (초) |
| `slam_reason` | string | SLAM 추적 상태 사유 |
| `remap_attempt` | int | TryRemap 시도 횟수 |
| `file_exists` | bool | 앵커 파일 존재 여부 (LoadFailed 시) |
| `in_sdk` | bool | SDK에 앵커 등록 여부 |

**분석 용도**: tracking robustness, recovery latency, environmental dependency — XR 논문 rebuttal 방어

### 2.4 Beam Pro Segments (`*_beam_segments.csv`)

```csv
segment_id,on_ts,off_ts,duration_s,mission_id,start_wp,end_wp,trigger_active,primary_tab,poi_view_count,info_card_open_count,comparison_count,mission_ref_count,zoom_count,map_view_duration_s
```

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `segment_id` | int | 세그먼트 순번 |
| `on_ts` / `off_ts` | ISO 8601 | ON/OFF 시각 |
| `duration_s` | float | 세그먼트 지속 시간 |
| `mission_id` | string | 세그먼트 시작 시 활성 미션 |
| `start_wp` / `end_wp` | string | 시작/종료 시 웨이포인트 |
| `trigger_active` | string | 세그먼트 시작 시 활성 트리거 |
| `primary_tab` | string | 첫 번째 전환 탭 |
| `poi_view_count` | int | POI 열람 횟수 |
| `info_card_open_count` | int | 정보 카드 열람 횟수 |
| `comparison_count` | int | 비교 카드 열람 횟수 |
| `mission_ref_count` | int | 미션 참조 열람 횟수 |
| `zoom_count` | int | 맵 줌 횟수 |
| `map_view_duration_s` | float | 맵 탭 체류 시간 |

**분석 용도**: "위기 지점에서 Beam 사용량 증가", "정보 탐색량과 confidence recovery 연관"

### 2.5 System Health (`*_system_health.csv`, 1Hz)

```csv
timestamp,fps,frame_time_ms,battery_level,battery_charging,thermal_status,memory_mb,tracking_state,slam_reason
```

| 컬럼 | 타입 | 설명 |
|------|------|------|
| `fps` | float | 초당 프레임 |
| `frame_time_ms` | float | 프레임 렌더링 시간 |
| `battery_level` | float | 배터리 잔량 (0-1) |
| `battery_charging` | string | 충전 상태 |
| `thermal_status` | string | 발열 상태 (none/light/moderate/severe/critical) |
| `memory_mb` | float | 할당 메모리 (MB) |
| `tracking_state` | string | SLAM 추적 상태 |
| `slam_reason` | string | SLAM 상태 사유 |

**분석 용도**: 논문 rebuttal 방어 — "성능 저하 때문에 결과가 달라진 것 아니냐" 질문 대응

**예상 크기**: 30분 기준 ~1,800행 (~180KB)
