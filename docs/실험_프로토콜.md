# 실험 프로토콜

## 1. 실험 설계 개요

| 항목 | 내용 |
|------|------|
| **설계** | 2 (기기 조건) × Within-subjects |
| **기기 조건** | Glass Only (G) / Hybrid (H) |
| **참가자 수** | 24명 (4그룹 × 6명) |
| **균형화** | 조건순서(2) × 미션 세트순서(2) 카운터밸런싱 |
| **소요 시간** | 참가자당 약 50분 |
| **장소** | 실내 건물 (단일 층, 복도 포함) |
| **장비** | XReal Air2 Ultra, XReal Beam Pro (Android), Unity 앱 |

## 2. 카운터밸런싱 배정표

### 2.1 4그룹 배정 (조건순서 × 미션 세트순서)

실험자가 메인 화면(AppModeSelector)에서 참가자 ID, 조건(Glass Only / Hybrid), 미션 세트(Set1 / Set2)를 직접 선택하여 단일 조건만 실행.

| 그룹 | 1차 실행 | 2차 실행 | 인원 |
|------|---------|---------|------|
| **G1** | Glass Only + Set1 | Hybrid + Set2 | 6명 |
| **G2** | Glass Only + Set2 | Hybrid + Set1 | 6명 |
| **G3** | Hybrid + Set1 | Glass Only + Set2 | 6명 |
| **G4** | Hybrid + Set2 | Glass Only + Set1 | 6명 |

### 2.2 참가자 배정 (24명)

| 참가자 | 그룹 | 1차 | 2차 |
|--------|------|-----|-----|
| P01~P06 | G1 | Glass Only + Set1 | Hybrid + Set2 |
| P07~P12 | G2 | Glass Only + Set2 | Hybrid + Set1 |
| P13~P18 | G3 | Hybrid + Set1 | Glass Only + Set2 |
| P19~P24 | G4 | Hybrid + Set2 | Glass Only + Set1 |

### 2.3 균형화 검증

- 각 조건(G, H)이 1차/2차에 동일 횟수(각 12회) 등장: OK
- 각 미션 세트(Set1, Set2)가 1차/2차에 동일 횟수(각 12회) 등장: OK
- 순서 효과 통제: G→H (12), H→G (12): OK
- 조건-세트 조합: 4가지(G+S1, G+S2, H+S1, H+S2) × 각 6회: OK

## 3. 경로 설계 가이드라인

### 3.1 경로 사양

| 항목 | 기준 |
|------|------|
| 웨이포인트 수 | 9개 (WP00 보정앵커 포함) |
| 미션 수 | 세트당 5개 (타입A×2, 타입B×2, 타입C×1) |
| 총 거리 | 약 100m |
| 예상 소요 시간 | 약 15분 |
| 층 구성 | **단일 층** (1개 층 내 이동) |
| 불확실성 트리거 | 세트당 2개 (4종을 Set1/Set2로 분배) |

### 3.2 불확실성 트리거 설계 (4종)

미션 세트당 2개의 불확실성 트리거를 배치:

**(T1) 안내 열화 — Tracking Degradation**
- AR 화살표가 5-8초간 떨림/깜빡임 후 사라짐 (tracking loss 시뮬레이션)
- 10초 후 자동 복구
- 참가자 경험: "내비게이션이 일시적으로 불안정"

**(T2) 정보 불일치 — Information Conflict**
- AR 화살표 방향과 환경 표지판 방향이 상충
- 실제로는 화살표가 정확 (표지판은 다른 목적지를 가리킴)
- 참가자 경험: "화살표와 표지판이 다른 방향"

**(T3) 해상도 부족 — Low Resolution Guidance**
- AR 화살표가 부채꼴(±30°)로 퍼져 모호한 방향만 표시
- 참가자 경험: "대략적 방향은 알겠는데 정확히 어디인지 모르겠다"

**(T4) 안내 부재 — Guidance Absence**
- 마지막 ~20m 구간에서 "목적지 근처입니다"만 표시, 화살표 제거
- Beam Pro 지도에는 정확한 핀 표시 유지
- 참가자 경험: "근처인 건 알겠는데 정확한 위치를 못 찾겠다"

**트리거 배치 매트릭스**:

| | Set1 | Set2 |
|---|------|------|
| WP03 부근 (미션 B1) | T2 (정보 불일치) | T1 (안내 열화) |
| WP06 부근 (미션 B2) | T3 (해상도 부족) | T4 (안내 부재) |

## 5. 실험자 스크립트

### 5.1 도입부

> "안녕하세요, 본 실험에 참여해 주셔서 감사합니다. 오늘 실험은 실내 길찾기에서 서로 다른 기기 사용 방식을 비교하는 연구입니다. 동일한 경로를 두 번, 각각 다른 기기 조건과 미션 세트로 수행하시게 됩니다. 각 회차에는 5개의 미션이 포함되어 있으며, 목적지를 찾아가서 특정 정보를 확인하는 과제를 수행하시게 됩니다. 각 회차는 약 11분 정도 소요되며, 전체 실험은 약 50분이 예상됩니다."

> "먼저 동의서를 읽어보시고, 질문이 있으시면 말씀해 주세요."

### 5.2 사전 설문

> "감사합니다. 이제 간단한 설문을 작성해 주세요. 평소 내비게이션 앱 사용 경험과 AR 기기 사용 경험에 대한 질문입니다."

### 5.3 각 조건 시작

> "이제 [N번째] 회차를 시작하겠습니다. 이번에는 [조건명] 조건입니다."

> "총 5개의 미션이 있습니다. 각 미션의 지시에 따라 목적지를 찾아가시고, 도착하면 화면에 나타나는 확인 질문에 답해 주세요. 또한 각 경유지에서 '현재 올바른 경로에 있다고 얼마나 확신하는가?'를 1(전혀 확신 없음)부터 7(매우 확신)까지 선택해 주세요."

> "준비되셨으면 시작하겠습니다."

### 5.4 경로 수행 중 (실험자 행동)

- 참가자로부터 약 3m 뒤에서 따라가며 관찰
- **개입하지 않음** (길을 잃어도 스스로 해결하도록 함)
- 단, 안전 위험 시 또는 5분 이상 같은 장소에서 못 찾을 경우 최소한의 힌트 제공
- 관찰 기록: 특이 행동, Beam Pro 참조 시점 (로그 데이터 보조)

### 5.5 조건 종료 후

> "수고하셨습니다. 이제 방금 경험에 대한 설문을 작성해 주세요."

> [NASA-TLX 배부] "이 설문은 방금 수행한 길찾기에서 느낀 부하(workload)에 대한 질문입니다."

### 5.6 종합 설문

> "두 조건 모두 완료하셨습니다. 마지막으로 두 가지 조건을 비교하는 설문을 작성해 주세요."

> "개방형 질문에서는 자유롭게 의견을 적어 주시면 됩니다. 특히 하이브리드 조건에서 Beam Pro의 어떤 정보(지도, POI 상세, 정보 카드, 비교 카드)가 가장 유용했는지, 어떤 상황에서 Beam Pro를 참조했는지, 자동으로 표시된 정보 카드가 도움이 되었는지에 대해 기억나는 점이 있으시면 알려 주세요."

## 6. 미션 설계 상세

> 미션 기반 길찾기의 상세 설계는 `실험_설계_v2.md` 참조. 아래는 실험자가 현장에서 참조할 요약.

### 6.1 미션 타입 요약

| 타입 | 명칭 | 수/세트 | 검증 방법 |
|------|------|---------|----------|
| A | 방향 안내 + 검증 | 2 | 목적지 도착 후 4지선다 질문 |
| B | 모호한 의사결정 | 2 | 올바른 방향 선택 자동 판정 |
| C | 정보 통합 | 1 | 통합 판단 4지선다 질문 |

### 6.2 미션 수행 중 실험자 행동

- 미션 브리핑이 화면에 표시되면 참가자가 읽는 것을 확인
- 미션 수행 중 추가 설명 제공하지 않음 (자력 해결 원칙)
- 검증 질문 응답 시 정답 여부를 참가자에게 알려주지 않음
- 단, 미션 이해 자체가 어려운 경우(문항 해석 오류) 최소한 명확화 가능

### 6.3 미션 브리핑 스크립트

> 아래 브리핑은 앱 UI에 텍스트로 자동 표시됨. 실험자가 구두로 읽어줄 필요 없음.

## 7. 조건별 기기 세팅 체크리스트

### 7.1 Glass Only 조건

- [ ] XReal Air2 Ultra 배터리 충분
- [ ] 글래스-Beam Pro 연결 확인 (미러링/앱 구동)
- [ ] AR 화살표 표시 정상 확인
- [ ] 글래스 위치 조정 (참가자 시야에 맞춤)
- [ ] 이벤트 로깅 시작 확인
- [ ] Beam Pro 화면 잠금 (참가자 접근 불가)

### 7.2 Hybrid 조건

- [ ] XReal Air2 Ultra 배터리 충분
- [ ] 글래스-Beam Pro 연결 확인
- [ ] AR 화살표 표시 정상 확인
- [ ] Beam Pro 정보 허브 대기 상태 (화면 OFF, 들면 ON)
- [ ] Beam Pro 화면 ON/OFF 감지 로깅 활성화 확인
- [ ] POI 데이터 정상 로드 확인 (모든 POI 마커 표시)
- [ ] 정보 카드 자동 트리거 설정 확인 (해당 미션 세트의 카드 데이터 로드)
- [ ] 비교 카드 데이터 로드 확인 (미션 C용)
- [ ] 정보 허브 인터랙션 로깅 활성화 확인 (7종 이벤트)
- [ ] 이벤트 로깅 시작 확인

### 7.3 공통 체크리스트

- [ ] 참가자 ID 및 조건 순서 확인
- [ ] 미션 세트 확인 (카운터밸런싱 배정표 대로)
- [ ] 시작 지점 위치 확인
- [ ] 확신도 입력 프롬프트 작동 확인
- [ ] 실험자 관찰 기록지 준비
- [ ] 비상 연락처 및 건물 출구 파악
