# 실험 프로토콜

## 1. 실험 설계 개요

| 항목 | 내용 |
|------|------|
| **설계** | 2 (기기 조건) × Within-subjects |
| **기기 조건** | Glass Only (G) / Hybrid (H) |
| **참가자 수** | 24명 (2그룹 × 12명) |
| **균형화** | AB/BA 카운터밸런싱 |
| **소요 시간** | 참가자당 약 50분 |
| **장소** | 실내 건물 (단일 층, 복도 포함) |
| **장비** | XReal Air2 Ultra, XReal Beam Pro (Android), Unity 앱 |

## 2. 카운터밸런싱 배정표

### 2.1 순서 배정 (S1/S2 × 홀짝 경로)

**조건 순서** (`CounterbalanceConfig.asset`에서 설정):

| 순서 그룹 | 1차 조건 | 2차 조건 |
|-----------|---------|---------|
| **S1** | Glass Only | Hybrid |
| **S2** | Hybrid | Glass Only |

**경로 순서** (참가자 번호로 암묵적 결정):

| 참가자 번호 | 1차 경로 | 2차 경로 |
|------------|---------|---------|
| **홀수** (P01, P03, ...) | Route A (서쪽-북쪽) | Route B (동쪽-북쪽) |
| **짝수** (P02, P04, ...) | Route B (동쪽-북쪽) | Route A (서쪽-북쪽) |

### 2.2 참가자 배정 (24명)

조건 순서(S1/S2) × 경로 순서(홀/짝)의 유연 배정 (고정 번호 범위 없음):

| 참가자 | 그룹 | 1차 | 2차 |
|--------|------|-----|-----|
| P01 (홀수, S1) | S1 | GlassOnly + Route A | Hybrid + Route B |
| P02 (짝수, S1) | S1 | GlassOnly + Route B | Hybrid + Route A |
| P03 (홀수, S2) | S2 | Hybrid + Route A | GlassOnly + Route B |
| P04 (짝수, S2) | S2 | Hybrid + Route B | GlassOnly + Route A |
| ... | ... | ... | ... |

> 실험 시작 시 `SessionSetupUI`에서 참가자 ID(P01~P24)와 순서 그룹(S1/S2)을 입력하면, 경로 순서(홀/짝)와 조건 순서가 자동 결정되어 미리보기로 표시됨.

### 2.3 균형화 검증

- 각 조건(G, H)이 1차/2차에 동일 횟수(각 12회) 등장: ✓
- 각 경로(A, B)가 1차/2차에 동일 횟수(각 12회) 등장: ✓ (홀/짝 각 12명)
- 순서 효과 통제: G→H (12), H→G (12) ✓
- 경로-조건 조합: 4가지(GA, GB, HA, HB) × 각 6회 ✓

## 3. 경로 설계 가이드라인

### 3.1 경로 사양

| 항목 | 기준 |
|------|------|
| 웨이포인트 수 | 8개 / 경로 |
| 미션 수 | 5개 / 경로 (타입A×2, 타입B×2, 타입C×1) |
| 총 거리 | 약 200m |
| 예상 소요 시간 | 약 15분 / 경로 |
| 층 구성 | **단일 층** (1개 층 내 이동) |
| 불확실성 트리거 | 2개 / 경로 (4종 순환 배치) |

### 3.2 불확실성 트리거 설계 (4종)

각 경로에 2개의 불확실성 트리거를 배치:

**(T1) 안내 열화 — Tracking Degradation**
- AR 화살표가 5-8초간 떨림/깜빡임 후 사라짐 (tracking loss 시뮬레이션)
- 10초 후 자동 복구
- 참가자 경험: "내비게이션이 일시적으로 불안정"

**(T2) 정보 불일치 — Information Conflict**
- AR 화살표 방향과 환경 표지판 방향이 상충
- 실제로는 화살표가 정확 (표지판은 다른 목적지를 가리킴)
- 참가자 경험: "화살표와 표지판이 다른 방향"

**(T3) 해상도 부족 — Low Resolution Guidance**
- AR 화살표가 부채꼴(±30°)로 퍼져 모호한 방향만 표시
- 참가자 경험: "대략적 방향은 알겠는데 정확히 어디인지 모르겠다"

**(T4) 안내 부재 — Guidance Absence**
- 마지막 ~20m 구간에서 "목적지 근처입니다"만 표시, 화살표 제거
- Beam Pro 지도에는 정확한 핀 표시 유지
- 참가자 경험: "근처인 건 알겠는데 정확한 위치를 못 찾겠다"

**트리거 배치 매트릭스**:

| | 경로 A | 경로 B |
|---|--------|--------|
| WP03 부근 | T1 | T2 |
| WP06 부근 | T4 | T3 |

## 6. 미션 설계 상세

> 미션 기반 길찾기의 상세 설계는 `실험_설계_v2.md` 참조. 아래는 실험자가 현장에서 참조할 요약.

### 6.1 미션 타입 요약

| 타입 | 명칭 | 수/경로 | 검증 방법 |
|------|------|---------|----------|
| A | 방향 안내 + 검증 | 2 | 목적지 도착 후 4지선다 질문 |
| B | 모호한 의사결정 | 2 | 올바른 방향 선택 자동 판정 |

### 6.2 미션 수행 중 실험자 행동

- 미션 브리핑이 화면에 표시되면 참가자가 읽는 것을 확인
- 미션 수행 중 추가 설명 제공하지 않음 (자력 해결 원칙)
- 검증 질문 응답 시 정답 여부를 참가자에게 알려주지 않음
- 단, 미션 이해 자체가 어려운 경우(문항 해석 오류) 최소한 명확화 가능

### 6.3 미션 브리핑 스크립트

> 아래 브리핑은 앱 UI에 텍스트로 자동 표시됨. 실험자가 구두로 읽어줄 필요 없음.
