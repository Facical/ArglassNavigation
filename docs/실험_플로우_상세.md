# 실험 모드 전체 흐름 상세

> 이 문서는 실험 모드의 전체 진행 흐름을 단계별로 정리합니다.
> 각 상태에서 **누가 무엇을 하면 → 어디에 무엇이 표시되는지** 한눈에 확인할 수 있습니다.

---

## 1. 3-Canvas 구조 요약

| Canvas | Render Mode | sortOrder | 대상 | 역할 |
|--------|------------|-----------|------|------|
| **ExperimentCanvas** | WorldSpace (head-locked) | 0 | 글래스 착용 피험자 | 미션 브리핑, 검증 질문, AR 화살표, HUD 등 |
| **BeamProCanvas** | 조건에 따라 동적 전환 | 1 | 피험자 (폰 or 글래스) | 3탭 정보 허브 (지도/정보카드/미션참조) |
| **ExperimenterCanvas** | ScreenSpaceOverlay | 10 | 실험자 (Beam Pro 폰) | 실험 제어 + 상태 모니터링 |

### BeamProCanvas 조건별 동작

| 조건 | Render Mode | 위치 | 입력 방식 | 핸드트래킹 |
|------|------------|------|----------|-----------|
| **GlassOnly** | WorldSpace | 글래스 head-locked (y=-0.15m, z=1.7m) | Hand Ray (핸드트래킹) | 활성화 |
| **Hybrid** | ScreenSpaceOverlay | Beam Pro 폰 화면 | 터치 | 비활성화 |

## 2. 전체 실험 흐름 다이어그램

```
앱 시작
  │
  ▼
[모드 선택 화면]  ──── 매핑 모드 (별도 흐름)
  │ "실험 모드" 선택
  ▼
[세션 설정] ── 참가자 ID + 순서 그룹 (S1/S2) 입력
  │ "시작" 버튼
  ▼
┌─── 매핑 데이터 있음? ───┐
│ Yes                      │ No
▼                          ▼
[Relocalization]     [Setup] (fallback 좌표)
│ 100%: 자동 전환
│ 부분실패: 실험자 판단
▼
[Setup] ── 세션 정보 확인
  │ "시작" 버튼
  ▼
[Condition1] ── 미션 5개 (A1→B1→A2→B2→C1)
  │ 전체 완료 후 자동 전환 (2초 딜레이)
  ▼
[Survey1] ── 조건 후 설문 (NASA-TLX, Trust Scale 등)
  │ "완료" 버튼
  ▼
[Condition2] ── 미션 5개 (반대 조건/경로)
  │ 전체 완료 후 자동 전환 (2초 딜레이)
  ▼
[Survey2] ── 조건 후 설문
  │ "완료" 버튼
  ▼
[PostSurvey] ── 사후 설문
  │ "완료" 버튼
  ▼
[Complete] ── 실험 종료
```

### 카운터밸런싱

| 그룹 | Condition1 | Condition2 |
|------|-----------|-----------|
| **S1** | GlassOnly | Hybrid |
| **S2** | Hybrid | GlassOnly |

| 참가자 번호 | 첫 경로 | 두번째 경로 |
|------------|---------|-----------|
| 홀수 (P01, P03...) | Route A | Route B |
| 짝수 (P02, P04...) | Route B | Route A |

## 3. 실험 상태별 상세 (ExperimentState)

### 3.1 Idle (앱 시작)

| 항목 | 내용 |
|------|------|
| **실험자가 할 일** | 모드 선택 화면에서 **"실험 모드"** 버튼 터치 |
| **글래스 화면** | GlassModeStatusPanel: "AR Navigation Experiment" + "Beam Pro에서 모드를 선택하세요" |
| **Beam Pro 화면** | AppModeSelectorPanel: 매핑 모드 / 실험 모드 버튼 2개 + 언어 선택 + 매핑 상태 표시 |
| **실험자 화면** | ExperimenterCanvas 표시되지만, ExperimentHUD는 Idle에서 비활성 |
| **전환 조건** | "실험 모드" 버튼 클릭 → SessionSetupPanel 표시 |
| **로깅** | 없음 |

### 3.2 Idle → 세션 설정 (SessionSetupUI)

> ExperimentState는 아직 Idle이지만, 실험 모드 선택 후 세션 설정 화면이 표시됩니다.

| 항목 | 내용 |
|------|------|
| **실험자가 할 일** | ① 참가자 ID 입력 (예: P01) ② 순서 그룹 선택 (S1/S2 드롭다운) ③ 프리뷰 확인 ④ **"시작"** 버튼 |
| **글래스 화면** | GlassModeStatusPanel 숨김 (Hide) |
| **Beam Pro 화면** | SessionSetupPanel: ID 입력 + 드롭다운 + 프리뷰 텍스트 + 시작 버튼 |
| **프리뷰 예시** | `P01 / S1 → Cond1: GlassOnly(Route A) → Cond2: Hybrid(Route B)` |
| **전환 조건** | "시작" 버튼 → `InitializeSession()` → 매핑 데이터 유무에 따라 Relocalization 또는 Setup |
| **로깅** | `EXPERIMENT_START` (order_group 포함) |

### 3.3 Relocalization

| 항목 | 내용 |
|------|------|
| **실험자가 할 일** | 피험자에게 주변을 둘러보도록 안내. 100% 성공 시 자동 진행. 부분 실패 시 **"Proceed"** 또는 **"Retry"** 버튼 선택 |
| **글래스 화면** | ExperimentHUD 활성화 (상태: Relocalization). 기타 글래스 UI 없음 (피험자는 주변을 둘러보기만) |
| **Beam Pro 화면** | RelocalizationPanel: 진행 바 + 성공/실패 카운트 + 상태 디테일 텍스트 |
| **실험자 화면** | ExperimenterHUD: 상태="Relocalization", Advance 버튼 활성 |
| **성공 시나리오** | 100% 성공 → "모든 앵커 인식 완료" → 1.5초 후 자동으로 Setup 전환 |
| **부분 실패** | ResultPanel 표시: 경고 텍스트(실패 웨이포인트 목록) + Proceed/Retry 버튼 + 10초 자동 Proceed 카운트다운 |
| **키보드 fallback** | `P`키: Proceed, `R`키: Retry (터치 불능 시 안전장치) |
| **전환 조건** | Proceed 클릭/자동 → `AdvanceState()` → Setup |
| **로깅** | `RELOCALIZATION_START`, `RELOCALIZATION_COMPLETE`, `RELOCALIZATION_PROCEED_PARTIAL` (부분 실패 시), `RELOCALIZATION_RETRY` |

### 3.4 Setup

| 항목 | 내용 |
|------|------|
| **실험자가 할 일** | 세션 정보를 확인하고 **"시작"** 버튼 클릭 |
| **글래스 화면** | ExperimentHUD: 상태="Setup" |
| **Beam Pro 화면** | ConditionTransitionPanel: 제목="실험 준비" + 참가자 ID/그룹 정보 + "시작" 버튼 |
| **실험자 화면** | ExperimenterHUD: 상태="Setup", Advance 버튼 활성 |
| **전환 조건** | "시작" 버튼 (transitionContinueButton) → `AdvanceState()` → Condition1 |
| **로깅** | 없음 (상태 전환은 Condition1 진입 시 기록) |

### 3.5 Condition1

| 항목 | 내용 |
|------|------|
| **진입 시 자동 동작** | ① `ConditionController.SetCondition()` 호출 → 핸드트래킹/BeamProCanvas 전환 ② `WaypointManager.LoadRoute()` → 경로 웨이포인트 로드 ③ ExperimentFlowUI가 ConditionTransitionPanel 표시 |
| **실험자가 할 일** | ① 조건/경로 정보 확인 ② **"시작"** 버튼으로 첫 미션 시작 |
| **글래스 화면** | ExperimentHUD: 상태="Condition1", 조건 표시 (GlassOnly/Hybrid) |
| **Beam Pro 화면** | 처음: ConditionTransitionPanel (조건명 + 경로명 + 디바이스 안내). "시작" 후: 3탭 정보 허브 활성화 |
| **실험자 화면** | ExperimenterHUD: 상태/조건/미션 정보 + Advance 버튼 + Next Mission 버튼(동적 라벨) |
| **전환 조건** | 미션 5개 전체 완료 → 2초 딜레이 → `AutoAdvanceExperiment()` → Survey1 |
| **로깅** | `CONDITION_CHANGE`, `ROUTE_START` |

**ConditionTransitionPanel 표시 내용:**
- 제목: "조건 1 시작"
- 디테일: 조건 라벨 (Glass Only / Hybrid) + 경로 (Route A / Route B)
- GlassOnly일 때: "글래스만 사용합니다. 핸드 제스처로 조작하세요."
- Hybrid일 때: "글래스 + 스마트폰을 함께 사용합니다."

### 3.6 Survey1

| 항목 | 내용 |
|------|------|

| **글래스 화면** | ExperimentHUD: 상태="Survey1" |
| **Beam Pro 화면** | SurveyPromptPanel: "설문 1 (조건명)" + 안내 텍스트 + "완료" 버튼 |
| **실험자 화면** | ExperimenterHUD: 상태="Survey1", Advance 버튼 활성 |
| **전환 조건** | "완료" 버튼 (surveyDoneButton) → `AdvanceState()` → Condition2 |
| **로깅** | `SURVEY_START` (survey: "post_condition_1") |

### 3.7 Condition2

| 항목 | 내용 |
|------|------|

## 4. 미션 사이클 상세 (MissionState)

Condition1/Condition2 동안 미션 5개가 순서대로 진행됩니다.
미션 순서: **A1 → B1 → A2 → B2 → C1** (각 Route 동일)

### 미션 타입

| 타입 | 설명 | 트리거 |
|------|------|--------|
| A_DirectionVerify | 방향 확인 (올바른 경로 선택) | T1 또는 T3 |
| B_AmbiguousDecision | 모호한 의사결정 (다중 정보 비교) | T2 또는 T4 |
| C_InfoIntegration | 정보 통합 (복합 판단) | 없거나 복합 |

### 4.1 Briefing (미션 브리핑)

| 항목 | 내용 |
|------|------|
| **자동 동작** | ① MissionBriefingUI 패널 표시 ② 트리거 활성화 (associatedTriggerId가 있으면) ③ BeamPro 데이터 로드 (MissionRef, InfoCards, POIs) |
| **글래스 화면** | BriefingPanel: 미션 ID + 브리핑 텍스트 + **"확인"** 버튼 |
| **Beam Pro 화면** | 3탭 허브에 미션 데이터 갱신 (MissionRef 탭에 미션 정보, InfoCard 탭에 정보카드, Map 탭에 관련 POI) |
| **실험자 화면** | Next Mission 버튼 라벨: **"Skip Briefing"** |
| **피험자 입력** | 글래스에서 "확인" 버튼 (핸드트래킹 또는 터치) |
| **전환 조건** | "확인" 클릭 → Navigation |
| **실험자 개입** | "Skip Briefing" 버튼 또는 `M`키 → `AdvanceBriefing()` → Navigation으로 강제 전환 |
| **로깅** | `MISSION_START` (mission_id, type, briefing) |

### 4.2 Navigation (내비게이션)

| 항목 | 내용 |
|------|------|
| **자동 동작** | AR 화살표 표시 (`ARArrowRenderer.Show()`) |
| **글래스 화면** | AR 화살표 (head-locked, 다음 웨이포인트 방향) + ExperimentHUD (현재 웨이포인트/거리) |
| **Beam Pro 화면** | 3탭 정보 허브 활성. 웨이포인트 도달 시 관련 InfoCard 자동 표시 (`TryAutoShow`) |
| **실험자 화면** | Next Mission 버튼 라벨: **"Force Arrival"** + 웨이포인트/거리 실시간 표시 |
| **전환 조건** | 목표 웨이포인트 도달 (WaypointManager.OnWaypointReached) → Arrival |
| **실험자 개입** | "Force Arrival" 버튼 또는 `M`키 → `ForceArrival()`, `J`키 → 웨이포인트 텔레포트 (에디터만) |
| **로깅** | `GLASS_ARROW_SHOWN`, `WAYPOINT_REACHED` (경유 웨이포인트), 트리거 관련 이벤트 |

#### 트리거 효과 (Navigation 중 발동)

| 트리거 | 효과 | 글래스 화면 변화 | 지속 시간 |
|--------|------|-----------------|----------|
| **T1: 트래킹 열화** | 화살표 ±5° 떨림(6초) → 화살표 소멸(3초) → 복귀 | 화살표 색상 주황색 전환 + 떨림 → 블랙아웃 | 약 9초 (자동 복귀) |
| **T2: 정보 충돌** | 물리적 안내판과 AR 안내 불일치 | 화살표 변화 없음 (물리 환경 트리거) | 미션 종료까지 |
| **T3: 저해상도** | 화살표가 부채꼴(±30°)로 확산 | 중앙 화살표 + 양쪽 3개씩 반투명 팬 화살표 | 미션 종료까지 |
| **T4: 안내 부재** | 화살표 완전 소멸 + 텍스트 표시 | "Near destination" 텍스트만 표시, 화살표 없음 | 미션 종료까지 |

### 4.3 Arrival (도착)

| 항목 | 내용 |
|------|------|
| **자동 동작** | 즉시 Verification으로 전환 (체류 시간 없음) |
| **로깅** | `MISSION_ARRIVAL` (waypoint_id, mission_id) |

### 4.4 Verification (검증 질문)

| 항목 | 내용 |
|------|------|
| **글래스 화면** | VerificationPanel: 질문 텍스트 + 답변 버튼 (2~4개 선택지) |
| **Beam Pro 화면** | 3탭 허브 유지 (피험자가 정보 참조 가능) |
| **실험자 화면** | Next Mission 버튼 라벨: **"Skip Mission"** |
| **피험자 입력** | 답변 버튼 중 하나 선택 |
| **전환 조건** | 답변 선택 → ConfidenceRating |
| **실험자 개입** | "Skip Mission" 버튼 또는 `M`키 → `ForceSkipMission()` → 다음 미션으로 건너뜀 |
| **로깅** | `VERIFICATION_ANSWERED` (answer, correct, rt_s) |

### 4.5 ConfidenceRating (확신도 평가)

| 항목 | 내용 |
|------|------|
| **글래스 화면** | ConfidenceRatingPanel: "답변에 대한 확신도" 프롬프트 + 1~7 버튼 + 확인 버튼 |
| **피험자 입력** | 1~7 중 하나 선택 → "확인" |
| **실험자 화면** | Next Mission 버튼 라벨: **"Skip Mission"** |
| **전환 조건** | 확인 → DifficultyRating |
| **로깅** | `CONFIDENCE_RATED` (rating 1-7) |

### 4.6 DifficultyRating (난이도 평가)

| 항목 | 내용 |
|------|------|
| **글래스 화면** | DifficultyRatingPanel: "길찾기 난이도" 프롬프트 + 1~7 버튼 + 확인 버튼 |
| **피험자 입력** | 1~7 중 하나 선택 → "확인" |
| **실험자 화면** | Next Mission 버튼 라벨: **"Skip Mission"** |
| **전환 조건** | 확인 → Scored |
| **로깅** | `DIFFICULTY_RATED` (rating 1-7) |

### 4.7 Scored (완료 / 다음 미션 대기)

| 항목 | 내용 |
|------|------|
| **자동 동작** | ① 트리거 해제 ② 2초 딜레이 후 다음 미션 자동 시작 (또는 전체 미션 완료 시 AdvanceState) |
| **글래스 화면** | 모든 미션 UI 닫힘. ExperimentHUD만 표시 |
| **전환 조건** | 남은 미션 있음 → 2초 후 `StartNextMission()` → Briefing. 마지막 미션 → 2초 후 → Survey |
| **로깅** | `MISSION_COMPLETE` (correct, duration_s), `TRIGGER_DEACTIVATED` |

### 미션 사이클 요약 다이어그램

```
StartNextMission()
  │
  ▼
[Briefing] ── 피험자 "확인" 또는 실험자 "Skip Briefing"
  │
  ▼
[Navigation] ── 화살표 따라 이동, 목표 WP 도달 또는 "Force Arrival"
  │
  ▼
[Arrival] ── (즉시 전환)
  │
  ▼
[Verification] ── 답변 선택
  │
  ▼
[ConfidenceRating] ── 1~7 선택 + 확인
  │
  ▼
[DifficultyRating] ── 1~7 선택 + 확인
  │
  ▼
[Scored] ── 2초 대기
  │
  ├── 남은 미션 → [Briefing] (다음 미션)
  └── 마지막 미션 → ExperimentState 자동 전환
```

## 5. GlassOnly vs Hybrid 조건 차이 요약표

| 항목 | GlassOnly | Hybrid |
|------|-----------|--------|
| **핸드트래킹** | 활성화 (Hand Ray) | 비활성화 |

| **BeamProUIAdapter** | 비활성화 (방향 감지 불필요) | 활성화 (Portrait/Landscape 전환) |
| **폰 화면 꺼짐 시** | 허브 계속 표시 (글래스 WorldSpace) | 허브 숨김 |

## 6. 실험자 긴급 개입 수단

### 6.1 ExperimenterHUD 버튼

| 버튼 | 동작 | 설명 |
|------|------|------|
| **Advance** | `ExperimentManager.AdvanceState()` | 현재 ExperimentState를 다음 단계로 강제 전환. 미션 진행 중이어도 상태 전환 가능 (주의!) |
| **Next Mission** | 미션 상태에 따라 동적 | 아래 표 참조 |
| **Capture Toggle** | `GlassViewCapture.ToggleCapture()` | 글래스 뷰 녹화 시작/중지 |

### 6.2 Next Mission 버튼 동적 라벨/동작

| 현재 MissionState | 버튼 라벨 | 동작 |
|-------------------|----------|------|
| **Briefing** | "Skip Briefing" | `AdvanceBriefing()` → Navigation 강제 전환 |
| **Navigation** | "Force Arrival" | `ForceArrival()` → Arrival 강제 전환 |
| **Arrival / Verification / ConfidenceRating / DifficultyRating** | "Skip Mission" | `ForceSkipMission()` → 모든 UI 닫고 다음 미션으로 |
| **Idle / Scored** | "Next Mission" | `StartNextMission()` → 다음 미션 시작 |

### 6.3 키보드 단축키 (에디터 테스트 + Relocalization)

| 키 | 동작 | 사용 가능 상태 |
|----|------|---------------|
| `N` | `AdvanceState()` — 실험 상태 다음 단계 | 항상 (에디터) |
| `M` | 미션 컨텍스트 제어 (Briefing→Skip, Navigation→ForceArrival, 기타→Skip/Start) | 항상 (에디터) |
| `J` | 현재 웨이포인트로 텔레포트 | Navigation 중 (에디터) |
| `B` | BeamProCanvas 토글 | 항상 (에디터) |
| `R` | 글래스 캡처 토글 | 항상 (에디터) |
| `P` | Relocalization Proceed | ResultPanel 활성 시 |
| `R` | Relocalization Retry | ResultPanel 활성 시 |
| `WASD` | 이동 | 항상 (에디터) |
| `Shift` | 달리기 | 항상 (에디터) |
| 우클릭 드래그 | 시점 회전 | 항상 (에디터) |
